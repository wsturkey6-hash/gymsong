# 怪獸訓練課程設計核心原則

> 這份檔案會被讀進 system prompt，作為 AI 排課表時的依據。先放一個基於《怪獸訓練肌力課程設計：打造最強壯版本的自己》（何立安）通用的起始版本；之後請從你的 PDF 補充更精確的章節摘要（例如：實際案例的週期化模板、特定動作的進階方式、deload 觸發條件等）。

---

## 一、訓練哲學

- **目標：打造最強壯版本的自己** — 訓練不是消耗，而是有目的的「漸進超負荷（progressive overload）」。
- **力量為一切之本** — 即使目標是肌肥大、肌耐力、爆發力，背後都需要紮實的肌力為基礎。
- **動作品質先於重量** — 寧可降一個強度區間，也不要為了堆數字而犧牲動作。

## 二、六大基本人類動作（必涵蓋）

每次完整課表應該涵蓋這六類其中的多數，避免長期失衡：

| 類別 | 中文 | 代表動作 |
|---|---|---|
| Squat | 蹲 | 背蹲舉、前蹲舉、高腳杯蹲 |
| Hinge | 髖屈伸 | 硬舉、羅馬尼亞硬舉、壺鈴擺盪 |
| Push | 推 | 臥推、肩推、伏地挺身 |
| Pull | 拉 | 引體向上、划船 |
| Core | 核心 | 棒式、抗旋轉、抗側屈 |
| Carry | 攜帶 | 農夫走路、公事包走路 |

## 三、強度區間（RM-based）

| 目標 | 反覆次數 (RM) | %1RM 約值 | 組數 | 組間休息 |
|---|---|---|---|---|
| 最大肌力 | 1-5 | 85-100% | 3-6 | 3-5 分鐘 |
| 肌肥大 | 6-12 | 67-85% | 3-5 | 1.5-3 分鐘 |
| 肌耐力 | 13+ | <67% | 2-4 | 30-90 秒 |
| 爆發力 | 1-5（高速） | 30-60%（動態） | 4-8 | 2-3 分鐘 |

## 四、週期化（Periodization）

- **線性週期**：每週固定增加重量/減少次數。適合初學者與中階。
- **波浪週期（Undulating）**：在同一週內穿插不同強度區間。適合中階以上。
- **共軛週期（Conjugate）**：同時訓練最大肌力與爆發力。適合進階。
- **典型 mesocycle**：4 週積累期 + 1 週 deload，或 3 週積累期 + 1 週 deload。
- **Deload 週**：強度降至 60-70%，組數量也減少約 30-50%，給神經系統恢復。

## 五、訓練週結構建議

### 每週訓練天數

| 天數 | 適合對象 | 建議分化 |
|---|---|---|
| 2-3 天 | 初學者 / 時間有限 | 全身訓練（whole body） |
| 3-4 天 | 中階 | 上下肢分化（upper/lower） |
| 4-5 天 | 進階 | 動作模式分化（push/pull/legs 或 squat/hinge/upper） |
| 5-6 天 | 高階 | 細部分化 + 弱點補強 |

### 每次訓練動作順序

1. **主要複合動作**（蹲/硬舉/臥推/肩推類）— 此時神經最清醒
2. **次要複合動作**（拉類、單側、變化式）
3. **輔助動作**（單關節、孤立、針對弱點）
4. **核心 / 攜帶 / 條件訓練**

## 六、進階模式（Progressive Overload）

- **重量進階**：每週加 2.5-5kg（下肢類）或 1-2.5kg（上肢類），維持次數不變
- **次數進階**：固定重量，每週多 1-2 下
- **組數進階**：固定重量與次數，增加總組數
- **密度進階**：縮短組間休息
- 初學者：以重量進階為主，每週可進步；中階：每 2-3 週進階；進階：每 mesocycle 進階

## 七、初學者起始重量原則

- 沒有 1RM 數據時，從**保守重量**起步（自體重的 0.5-1 倍視動作而定）
- 第一週只做技術組（technique sets），確認動作品質
- 第二週開始才嘗試系統性加重
- 主要動作（蹲、硬舉、臥推、肩推）建議：男性中階起步約自體重 60-100% 範圍視動作而定，女性 40-60%

## 八、安排課表時的檢查清單

排出一份課表後，AI 應自我檢查：

- [ ] 涵蓋至少 4 類基本動作（蹲/髖/推/拉至少都要有）
- [ ] 每週訓練間隔合理（同肌群至少間隔 48 小時）
- [ ] 強度區間與目標一致
- [ ] 每 4-6 週有 deload 週
- [ ] 主要動作排在訓練前段
- [ ] 沒有超出使用者「可用器材」清單的動作
- [ ] 對於初學者，主要動作每週進階；中階以上週期化進階

## 九、規範與限制

- 任何選用的動作 ID 必須來自使用者提供的動作庫（available_exercises）
- 若使用者只有自體重器材，避免推薦需要重量的動作
- 若使用者標註受傷部位，避免該部位的高風險動作
- 重量建議須保守，第一週可標 `null` 讓使用者自己試
